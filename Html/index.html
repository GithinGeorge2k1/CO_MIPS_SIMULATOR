Team Instinct Simulator

This Simulator is built as part of our Computer Organization Course and mimics a subset of what the mips processor can do. This Application was developed in 3 phases and in each phase we've added the following components of Mips processor.

  1. A Basic Mips Simulator With a subset of the mips instruction set
  2. Pipeline To The Mips Processor
  3. Two Levels of data Cache With LRU Replacement Policy - (BlockSize set to same)

We Have Learned a lot Of Ideas regarding Mips Processor and Computer Organization in General. Through this application we have to tried to realize a few of the Mips features as correctly as possible.


Following are the instructions for using this application:


  • The Functionality "Reinitialize and load File" is used to load any text format file containing assembly code. The code is then parsed and proceeded to execute. If any errors were found then the file is not opened and "Run" function will be disabled. The Cache parameters are provided at this stage. All parameters except latency must be power of 2 and blockSize must be greater than or equal to 4.
  • The "Run" and "Run Step By Step" functions correspond to their meanings. If the applications recognizes any valid code at present, the user can run the assembly code.
  • The user can enable Operand Forwarding by clicking on Enable Forwarding option in Status Bar. One can disable the same feature by the clicking the same option once more.

The Application has the following tabs:

  1. Registers Tab - Shows all the registers and their current Values
  2. Data Tab - Shows all global and static variables declared in the program
  3. Text Tab - Shows the formatted assembly code on screen with their corresponding machine code instructions and labels(if any)
  4. Timeline Tab - This Tab contains a table which represents the timeline of the program. There is a cutoff Limit for the table after 2000 instructions are issued.(We are not sure about the correctness of the Timeline - But it is to our knowledge)
  5. Stall Tab - Lists all instructions having some stall with their corresponding No Of Stalls
  6. Console - Displays relevant insformation to the user(we have not included system calls in the pipeline, hence user can't take input)

About the processor Design:-

The Simulator is of a 5 stage pipeline with Branch Not Taken predictor and Write Back, Write Allocate Cache

There is no support for Stack and Stack Pointer as of Yet.

There are no support for pseudo instructions and syscalls. Hence a console for taking input and printing output is not present

The machine is word addressable Only. Byte addressable instructions were not realised. As a result Block Size in cache should also be a multiple of the word size(Word Size = 4 bytes)

Also the cache is realized in simulator with the assumption that there is a store queue or a buffer that writes to Main Memory . While writing to MM, we assumed that all memory read concerning that address is taken from the store queue.

Also, keeping data into cache is not considered in the Clock Cycle Count

Latency of the two memory operations can be set for L1 and L2 cache. But For MM it is defaultly set to 100 Cycles

Inclusive Caching policy is used in our design. ie L2 is inclusive of L1
