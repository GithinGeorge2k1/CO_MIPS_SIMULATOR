A Quick Summary On Mips: MIPS (Microprocessor without Interlocked Pipelined Stages)[1] is a reduced instruction set computer (RISC) instruction set architecture (ISA)[2]:A-1[3]:19 developed by MIPS Computer Systems, now MIPS Technologies, based in the United States. There are multiple versions of MIPS: including MIPS I, II, III, IV, and V; as well as five releases of MIPS32/64 (for 32- and 64-bit implementations, respectively). We Have Learned a lot Of Ideas regarding Mips Processor and Computer Organization in General. Through this application we have to tried to realize a few of the Mips features as correctly as possible.

Following are the instructions for using this application:

  • The Functionality "Reinitialize and load File" is used to load any text format file containing assembly code. The code is then parsed and proceeded to execute. If any errors were found then the file is not opened and "Run" function will be disabled
  • The "Run" and "Run Step By Step" functions correspond to their meanings. If the applications recognizes any valid code at present, the user can run the assembly code.
  • The user can enable Operand Forwarding by clicking on "Enable Forwarding" option. One can disable the same feature by the clicking the same option once more. Initially Forwarding is Not Enabled.

The Application has the following tabs:

  1. Registers Tab - Shows all the registers and their current Values
  2. Data Tab - Shows all global and static variables declared in the program
  3. Text Tab - Shows the formatted assembly code on screen with their corresponding machine code instructions and labels(if any)
  4. Timeline Tab - This Tab contains 3 tables which represents the timeline of the program. There is a cutoff Limit for the table after 3000 instructions are issued.(We are pretty sure we have made some mistakes in making the timeline diagram)
  5. Stall Tab - Lists all instructions having some stall with their corresponding No Of Stalls
  6. Console - Displays relevant insformation to the user(we have not included system calls in the pipeline, hence user can't take input)
